# 출시 전 최종 점검 결과

점검일: 2026-07-17 · 대상: `얼마 있어?` (budgettrip)

심각도 기준
- **치명**: 출시 전 반드시 고쳐야 함 (사용자에게 앱이 망가진 것처럼 보이거나, 필수 리소스 누락)
- **중요**: 출시는 가능하지만 리스크가 실재함 (비용 남용, 심사 반려 가능성, 회귀 안전망 부재)
- **경미**: 알아두면 좋지만 지금 당장 막을 필요는 없음

## 우선순위별 이슈 목록

### 🔴 치명

1. **React Error Boundary가 없음** (`src/main.jsx`)
   렌더링 중 어디서든 예외가 발생하면 앱 전체가 흰 화면으로 죽고, 사용자가 복구할 방법이 없음(새로고침 외엔 없음). 특히 "저장한 코스"는 `localStorage`에 스키마 검증 없이 저장·복원되는데, 이번 세션에서 저장 데이터 구조(장소 role, 비용 필드 등)가 여러 번 바뀌었어서 예전 버전에서 저장된 코스를 새 버전에서 열면 예상 못한 형태로 렌더링을 시도할 가능성이 있음.
   → 최상위에 Error Boundary를 추가해 "문제가 발생했어요, 다시 시도해주세요" 같은 복구 UI로 감싸는 걸 권장.

2. **앱 아이콘 미설정** (`granite.config.ts` — `brand.icon: ''`)
   세션 초반부터 TODO로 남아있던 항목이 아직 비어있음. 이 상태로 마켓에 등록하면 아이콘 없이 노출됨.
   → 콘솔에 등록한 아이콘 이미지 URL을 채워야 함.

### 🟠 중요

3. **`/api/ai-plan`, `/api/tour-places`에 요청 빈도 제한(rate limiting)이 전혀 없음** (`server/app.js`)
   "하루 3회 무료" 제한은 브라우저 `localStorage` 기반이라, API를 직접 호출하는 스크립트는 클라이언트를 거치지 않고 바로 서버를 두드릴 수 있음. 특히 `/api/ai-plan`은 호출마다 실제 OpenAI 과금이 발생해서, URL이 알려지면 과금 남용에 취약함.
   → IP 기준 또는 간단한 토큰버킷 rate limit 추가 권장.

4. **디버그용 엔드포인트가 인증 없이 프로덕션에 노출됨** (`/api/kakao-health`, `server/app.js:111`)
   누구나 호출 가능하고, 호출할 때마다 실제 카카오 API 쿼터를 소모하며, 카카오 키의 "존재 여부/길이" 메타데이터를 노출함(키 값 자체는 아님).
   → 배포 전 라우트 제거 또는 최소한 접근 제한 권장.

5. **개인정보처리방침 링크가 앱 안 어디에도 없음**
   광고 SDK(TossAds/AdMob 미디에이션)를 쓰고 있어 기기·광고 식별자 수집이 있을 수 있는데, 이에 대한 안내가 앱 내 어디에도 없음. 스토어 심사에서 요구될 가능성이 높음.
   → 결과 화면 하단 등에 개인정보처리방침 링크 추가 권장.

6. **자동화 테스트가 전혀 없음**
   테스트 프레임워크 미설치, 테스트 파일 0개. 로그인·결제 기능은 없어서 해당 항목 자체는 없지만, 예산 계산·코스 생성·인원수 반영 같은 핵심 로직이 전부 수동 테스트로만 검증돼 있어 향후 수정 시 회귀를 잡아줄 안전망이 없음.
   → 최소한 `budget.js`(예산 계산)처럼 순수 함수 위주로 핵심 로직에 대한 단위 테스트를 붙이는 걸 권장.

### 🟡 경미

7. **`ait build`가 로컬 웹 서빙용 `dist/`를 덮어씀**
   `npx ait build`가 내부적으로 `dist/`를 RN 번들 출력용으로 재사용해서, 그 직후 `node server/app.js`로 로컬 확인하면 503이 남(이번 세션에서 실제로 "PC에서 화면 안 나와요"로 재현됨). `.ait` 파일 자체엔 문제 없음 — `npm run build`로 다시 복구하면 됨. 배포 스크립트 순서에 주의 메모만 남겨두면 됨.

8. **`courses` 계산이 예산 입력 중에도 매 키 입력마다 재계산됨** (`src/App.jsx`)
   `useMemo`가 `input` 객체 전체를 의존성으로 써서, 예산 필드에 숫자 하나 입력할 때마다 코스 생성 로직이 다시 돎(화면엔 안 보여도). 네트워크 호출이 아니라 체감 랙은 거의 없음.

9. **`render.yaml`이 실제 배포 설정과 다름**
   실제로는 Render 콘솔에서 수동으로 설정했고(서비스명 `budgettrip-api`), 이 파일엔 `name: summer-travel`로 남아있고 `ALLOWED_ORIGINS`/`VITE_API_BASE_URL`도 빠져있음. 지금 안 쓰이는 파일이라 기능엔 영향 없지만, 나중에 누가 이 파일을 참고하면 헷갈릴 수 있음.

## ✅ 문제 없음으로 확인된 항목

- **클린 빌드**: `dist/` 삭제 후 재빌드 성공, 프로덕션 서버(`node server/app.js`) 정상 기동
- **전체 사용자 플로우**: 스플래시→입력→성향테스트→결과→코스생성까지 실제 브라우저로 재현, 콘솔 에러 0건
- **하드코딩된 시크릿 없음**: 전체 git 커밋 히스토리(diff 포함) 검색 완료, 실제 키 값 노출 이력 없음
- **`.gitignore`**: `.env`/`.env.local`/`.env.*.local` 모두 포함, 실제로 한 번도 커밋된 적 없음
- **네트워크/API 실패 처리**: 서버·클라이언트 양쪽 다 `try/catch` + graceful fallback, 로딩 화면 12초 타임아웃 안전장치 있음
- **메모리 누수**: `setInterval`/`addEventListener`/`ResizeObserver` 전부 정상 cleanup 확인
- **번들 크기**: JS 82KB(gzip), 전체 에셋 약 490KB — 앱 규모 대비 양호
- **환경변수 분리**: 로컬(`.env.local`)과 프로덕션(Render 콘솔 직접 입력) 분리 잘 돼 있음
- **권한 요청**: `permissions: []` — 위치/카메라 등 미사용, 별도 설명 불필요
