## 우선순위별 이슈 목록 (전부 ✅ 처리 완료)

### 🔴 치명

1. ✅ **React Error Boundary가 없음** (`src/main.jsx`)
   → `src/ErrorBoundary.jsx` 신규 추가, `main.jsx`에서 `<App />`을 감쌈. "다시 시도" / "저장한 코스
   초기화 후 다시 시도" 두 가지 복구 옵션 제공. Playwright로 의도적으로 깨진 저장 코스 데이터
   (`course: null`)를 주입해 실제 예외를 유발시켜 검증 — 폴백 UI 정상 노출, 초기화 버튼으로
   정상 복구까지 확인.

2. ✅ **앱 아이콘 미설정** (`granite.config.ts` — `brand.icon: ''`)
   → `public/icon.png` 생성(브랜드 teal 컬러 + 여행가방 + ₩ 모티프), `brand.icon`을
   `https://budgettrip-api.onrender.com/icon.png`로 채움. 로컬 서버로 200 서빙 확인.
   **주의**: 앱인토스 콘솔의 마켓 등록 화면에도 이 아이콘 이미지를 별도로 업로드해야 함(이건
   콘솔 작업이라 사용자가 직접 진행 필요).

### 🟠 중요

3. ✅ **`/api/ai-plan`, `/api/tour-places`에 요청 빈도 제한이 전혀 없음**
   → `server/app.js`에 IP 기준 슬라이딩 윈도우 rate limit 추가(`/api/ai-plan` 10분당 10회,
   `/api/tour-places` 10분당 60회, 초과 시 429). 동시 요청 65개로 실제 429 발생 확인, 이후 정상
   플로우가 방해받지 않는 것도 확인. 오래된 IP 버킷은 30분마다 자동 정리.

4. ✅ **디버그용 `/api/kakao-health`가 인증 없이 프로덕션에 노출됨**
   → 라우트 및 `diagnoseKakaoLocal` 함수 완전히 제거. 호출 시 404 확인.

5. ✅ **개인정보처리방침 링크가 앱 안 어디에도 없음**
   → `server/app.js`에 `/privacy` 정적 페이지 추가(수집 데이터 없음/localStorage 사용/광고 SDK/
   카카오맵 안내 + 문의처). 스플래시 화면 맨 아래에 작은 링크로 연결. 200 서빙 확인.

6. ✅ **자동화 테스트가 전혀 없음**
   → `vitest` 설치, `src/lib/budget.test.js`에 23개 단위 테스트 작성(예산 계산 전반 +
   인원수(party) 반영 회귀 테스트 포함). `npm test`로 실행, 전부 통과.
   - 테스트 작성 중 부수 발견: `parseCostString`이 "5천~1.2만원"처럼 **단위가 섞인 범위 문자열**은
     의도한 대로 파싱하지 못함(마지막 단위를 양쪽에 적용). 다만 실제 코드에서 이 함수가 호출되는
     경로는 전부 같은 단위끼리의 범위 문자열만 쓰고 있어서 **현재는 도달하지 않는 잠재 버그**임 —
     당장 고칠 필요는 없지만 향후 가격 문구를 바꿀 때 참고할 것.

### 🟡 경미

7. ✅ **`ait build`가 로컬 웹 서빙용 `dist/`를 덮어씀**
   → `package.json`에 `"ait:build": "ait build && npm run build"` 스크립트 추가 — `.ait` 생성
   직후 자동으로 `dist/`를 웹 서빙용으로 복구함. `TOSS_LAUNCH.md`에도 이 스크립트를 쓰라고 안내
   업데이트. 실제로 실행해서 `.ait` 생성 + `dist/index.html` 정상 복구 둘 다 확인.

8. ✅ **(재검토 후 변경 없음) `courses` 계산이 예산 입력 중 매 키 입력마다 재계산됨**
   → 실측 결과 `generateCourses` 1회 호출에 평균 0.065ms(60fps 프레임 예산의 1/250 수준) — 체감
   불가능한 수준이라 코드 변경 없이 종결. 괜히 의존성 배열을 쪼개는 게 오히려 불필요한 복잡도였음.

9. ✅ **`render.yaml`이 실제 배포 설정과 다름**
   → 서비스명을 `budgettrip-api`, 리전을 `singapore`로 맞추고 `VITE_API_BASE_URL`/
   `ALLOWED_ORIGINS` 환경변수를 추가. 실제 배포는 콘솔에서 수동으로 했다는 안내 주석도 남김.

## ✅ 원래부터 문제 없음으로 확인된 항목

- **클린 빌드**: `dist/` 삭제 후 재빌드 성공, 프로덕션 서버(`node server/app.js`) 정상 기동
- **전체 사용자 플로우**: 스플래시→입력→성향테스트→결과→코스생성까지 실제 브라우저로 재현, 콘솔 에러 0건
- **하드코딩된 시크릿 없음**: 전체 git 커밋 히스토리(diff 포함) 검색 완료, 실제 키 값 노출 이력 없음
- **`.gitignore`**: `.env`/`.env.local`/`.env.*.local` 모두 포함, 실제로 한 번도 커밋된 적 없음
- **네트워크/API 실패 처리**: 서버·클라이언트 양쪽 다 `try/catch` + graceful fallback, 로딩 화면 12초 타임아웃 안전장치 있음
- **메모리 누수**: `setInterval`/`addEventListener`/`ResizeObserver` 전부 정상 cleanup 확인
- **번들 크기**: JS 82KB(gzip), 전체 에셋 약 490KB — 앱 규모 대비 양호
- **환경변수 분리**: 로컬(`.env.local`)과 프로덕션(Render 콘솔 직접 입력) 분리 잘 돼 있음
- **권한 요청**: `permissions: []` — 위치/카메라 등 미사용, 별도 설명 불필요

## 참고: npm audit 관련

`vitest` 설치 후 `npm audit`에서 31개 취약점(critical 1건 포함)이 나오는데, 전부
`@apps-in-toss/*`/`@granite-js/*` **빌드 도구 체인**(babel, fastify, esbuild 등) 안에 있는
것들이고 패치도 아직 없음. 이 도구들은 `.ait` 빌드 시 개발자 로컬에서만 실행되고, 실제 배포되는
Render 서버나 브라우저 번들에는 포함되지 않음 — 당장 조치 불필요.
