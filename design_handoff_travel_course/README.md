# Handoff: 지역·예산 기반 AI 여행 코스 추천 미니앱

## Overview
앱인토스(토스 인앱 웹뷰) 미니앱. 사용자가 지역·예산·이동수단을 입력하고 5문항 성향 테스트를 거치면, 소비 성향(숙박/식비/관광지 비중)에 맞춰 OpenAI API가 여행 코스 3개를 생성해 카드로 보여준다. 이 핸드오프는 **6개 화면의 UI 시안**을 실제 코드로 옮기기 위한 문서다.

플로우: **스플래시 → 입력 → 성향 테스트(5문항) → 성향 결과 → 로딩(코스 생성) → 결과**

## About the Design Files
이 번들의 HTML 파일은 **HTML로 만든 디자인 레퍼런스(프로토타입)** 이다. 의도한 룩앤필과 동작을 보여줄 뿐 그대로 복붙할 프로덕션 코드가 아니다. 목표는 이 시안을 **대상 코드베이스(React + Vite + Tailwind CSS)에 그 환경의 관행대로 재구현**하는 것이다.

- `여행코스추천 미니앱 시안.dc.html` 은 "Design Component" 형식(스트리밍 프리뷰용 커스텀 런타임)으로 작성돼 있다. **런타임/문법을 옮기지 말고, 화면의 시각 결과물만** 참고하라. 인라인 style로 되어 있으므로 값(색/여백/폰트)을 그대로 읽어 Tailwind 클래스나 토큰으로 치환하면 된다.
- iOS 폰 프레임(`ios-frame.jsx`)은 시안을 폰 안에서 보여주기 위한 목업 껍데기다. **실제 제품에는 넣지 않는다** — 토스 웹뷰가 이미 기기 크롬을 제공한다. 각 화면의 "내용"만 구현하라.

## Fidelity
**High-fidelity (hifi).** 최종 색상·타이포·간격·인터랙션이 확정된 픽셀 목업이다. 아래 스펙과 디자인 토큰을 그대로 사용해 픽셀에 가깝게 재현하되, 스타일링은 Tailwind로 표현한다.

## Product Logic (별도 문서 참조)
예산 계산, 성향 점수 로직, 당일치기 분기, TourAPI/OpenAI 프롬프트 설계 등 **기능 로직은 기획서 `여행코스추천_미니앱_기획서_v1.0.md` 의 3~4절이 단일 진실 소스(SSOT)** 다. 이 README는 화면/시각/인터랙션만 다룬다. 구현 시 두 문서를 함께 볼 것.

- 상단은 토스 웹뷰 상태바를 피하려 콘텐츠 시작 `padding-top: 60px`.

## Interactions & Behavior
- **화면 전환**: 스플래시 시작하기 → 입력. 입력 CTA → 테스트. 테스트 완료 → 성향 결과. 결과 CTA(내 맞춤 코스 보기) → 로딩 → (API 완료) 결과.
- **입력 화면**: 지역 미선택/예산 미입력 등 필수 미충족 시 하단 CTA 비활성(§3.1 검증 규칙). 지역 확정 즉시 TourAPI areaBasedList **백그라운드 프리페치**(§3.1) — 테스트 진행 중 완료되도록.
- **성향 테스트**: 선택지 탭 → 답 저장·카드 선택 표시·진행률 갱신. 문항 전환 시 페이드/슬라이드 애니메이션(§11.3).
- **로딩**: 스피너 `spin 0.95s`, 내부 원 bob 1.8s. 실제 처리 순서는 §3.3.
- **결과**: 세그먼트 탭 클릭 → 활성 코스 전환(라벨/제목/비용/비율 바/장소 리스트/동선/지도 핀이 해당 코스 데이터로 교체). 공유 버튼 → 앱인토스 SDK 공유.
- **hover(웹뷰 데스크톱 테스트용)**: 테스트 선택지 `translateY(-2px)`.

## State Management (UI 관점)
- `input`: { regionCode, region, period('당일치기'|'1박2일'|'2박3일'), party:number, budget:number, transit('자차'|'대중교통'), fareIncluded:boolean }
- `testAnswers`: 문항별 선택(A/B/C) → §3.2 배점으로 L/F/A 합산 → 비율/라벨 산출
- `personality`: { label, ratios:{stay,food,sight} }
- `courses`: OpenAI 응답 파싱 결과(코스 3개). 각 코스 = { label, accent, title, budget, ratios, transit, places:[{no, kind, icon, name, tag, cost}] }
- `activeTab`: 0(메인)/1/2 — 결과 세그먼트
- `loading`: boolean; 데이터 페칭: TourAPI(프리페치) + OpenAI(코스 생성). 동일 지역·예산·성향 조합 캐싱 고려(리스크 §9).

## Assets
- **폰트**: Pretendard(CDN, 위 토큰 참조).
- **아이콘**: 뒤로 chevron, 돋보기, ±, 체크, 공유, 축 색 점 등은 인라인 SVG(단순 도형) — 코드베이스 아이콘 세트로 대체 가능.
