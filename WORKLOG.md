## 2026-07-06 — 카카오맵 미지원 장소 노출 방지

TourAPI/Kakao 실데이터가 있는 결과 화면에서도 날짜별 장소 수를 채우는 보충 후보(`강릉 동네 카페`, `강릉 무료 전시관` 등)가 실제 장소처럼 노출될 수 있는 문제를 수정했다.

- **`src/lib/courses.js`**: 카카오 장소 ID와 실제 `place.map.kakao.com` URL이 있는 장소만 실시간 코스 후보로 사용하도록 필터링.
- 실시간 검증 장소가 충분할 때는 일반 보충 후보(`extraPlaces`)를 날짜별 코스에 섞지 않도록 변경.
- 2일차/3일차 장소 채우기는 검증된 TourAPI+Kakao 후보 풀을 우선 사용하도록 개선.
- **`src/App.jsx`**: "바꾸기" 후보 목록도 카카오 검증 장소만 표시하도록 방어.
- 결과: 카카오맵에서 지원하지 않는 일반 문구형 장소가 결과 화면과 교체 후보에 표시되지 않음.

### 추가 보강
- **`server/kakaoLocal.js`**: 카카오 Local에서 실제 장소 ID/상세 URL을 찾지 못한 경우 검색 URL을 대신 넣지 않고 `kakaoSupported: false`로 표시.
- **`src/App.jsx`**: 결과 화면 목록/지도/예산 계산에서 카카오 검증 실패 장소를 제외하고, 검증 장소가 없으면 안내 문구 표시.
- **`vite.config.js`**: `npm run dev`(`localhost:5173`)에서도 `/api/*`가 로컬 API 서버(`127.0.0.1:5175`)로 연결되도록 프록시 추가.
- 결과: `map.kakao.com/link/search/...` 검색 결과 없음 페이지로 이동하는 장소 링크를 화면에서 제거.

## 2026-07-06 — 성향 결과 이미지 잘림 수정

성향 결과 화면의 일러스트가 원형 마스크(`rounded-full`)와 `object-cover`로 다시 잘리면서, 이미 둥근 사각형으로 제작된 이미지의 가장자리 요소가 답답하게 잘려 보이는 문제를 수정했다.

- **`src/App.jsx`**: 성향 결과 이미지 스타일을 원형 크롭에서 둥근 사각형(`rounded-[30px]`) + `object-contain`으로 변경.
- 결과: 미식/호캉스/알뜰 성향 일러스트가 원본 비율과 모서리 형태를 유지하며 표시되고, 음식·소품 가장자리 잘림이 사라짐.

### 1. 결과 화면 모바일 재구성 (`src/App.jsx`)
- 코스 카드 높이 **1,874px → 830px (약 56%↓)**, 실제 코스(장소 리스트)가 첫 화면에 노출되도록 순서 재배치.
- 시안에 없던 중복 블록 **"AI 추천 흐름(슬롯)" 제거** (아래 실제 장소 리스트와 내용 중복이었음).

### 2. 이미지 경량화 (`src/assets/`)

### 3. 외부 API 3종 검증 (실제 키로 호출)
