# 작업 로그

## 2026-07-06 — 토스 인앱 모바일 레이아웃 1차 최적화

토스 인앱 웹뷰 서비스 준비를 위해 모바일 화면 기준으로 전체 앱 껍데기와 결과 화면 밀도를 정리했다.

- **`src/App.jsx`**: 앱/입력/결과 화면 높이를 `100dvh` 기준으로 정리해 모바일 웹뷰에서 화면이 흔들리거나 바깥 스크롤이 생기는 문제를 줄임.
- 하단 고정 CTA에 `safe-area-inset-bottom`을 반영하고, 배경 블러/여백을 조정해 토스 하단 제스처 영역과 겹치지 않도록 개선.
- 입력 화면 필드 간격, 칩 높이, 예산 입력 높이를 모바일 기준으로 축소해 작은 화면에서도 핵심 입력이 더 많이 보이도록 조정.
- 결과 화면 카드 패딩, 제목/AI 요약/예산 미터/일차 탭/이동 안내 간격을 압축해 장소 리스트 진입까지의 거리를 줄임.
- 장소 행의 아이콘·텍스트·비용 표기를 조밀하게 정리하되, "바꾸기" 버튼은 28px 높이로 유지해 터치 가능성을 확보.
- **`src/index.css`**: `100dvh`, 터치 하이라이트 제거, overscroll 방지 등 모바일 웹뷰 전역 스타일 보강.
- 검증: 390×844 및 360×740 모바일 뷰포트에서 입력/성향/결과 플로우 확인. 가로 넘침 없음.

### 로딩 연출 보강
- **`src/App.jsx`**: 코스 생성 로딩 화면 중앙의 임시 `SUMMER` 배지를 실제 여행 일러스트 이미지로 교체.
- 스피너 링은 유지하되 내부 이미지를 둥근 사각형 카드로 표시해 빈 화면처럼 보이는 문제를 개선.

### 지도 SDK 로딩 안정화
- **`src/App.jsx`**: Kakao Maps JavaScript SDK 주소를 프로토콜 상대 URL(`//dapi...`)에서 `https://dapi...`로 고정.
- 로컬 개발 서버가 `localhost:5174`처럼 다른 포트로 뜰 경우 카카오 콘솔 Web 도메인 등록이 필요함을 확인.

## 2026-07-06 — 카카오맵 미지원 장소 노출 방지

TourAPI/Kakao 실데이터가 있는 결과 화면에서도 날짜별 장소 수를 채우는 보충 후보(`강릉 동네 카페`, `강릉 무료 전시관` 등)가 실제 장소처럼 노출될 수 있는 문제를 수정했다.

- **`src/lib/courses.js`**: 카카오 장소 ID와 실제 `place.map.kakao.com` URL이 있는 장소만 실시간 코스 후보로 사용하도록 필터링.
- 실시간 검증 장소가 충분할 때는 일반 보충 후보(`extraPlaces`)를 날짜별 코스에 섞지 않도록 변경.
- 2일차/3일차 장소 채우기는 검증된 TourAPI+Kakao 후보 풀을 우선 사용하도록 개선.
- **`src/App.jsx`**: "바꾸기" 후보 목록도 카카오 검증 장소만 표시하도록 방어.
- 결과: 카카오맵에서 지원하지 않는 일반 문구형 장소가 결과 화면과 교체 후보에 표시되지 않음.

### 추가 보강
- **`server/kakaoLocal.js`**: 카카오 Local에서 실제 장소 ID/상세 URL을 찾지 못한 경우 검색 URL을 대신 넣지 않고 `kakaoSupported: false`로 표시.
- **`src/App.jsx`**: 결과 화면 목록/지도/예산 계산에서 카카오 검증 실패 장소를 제외하고, 검증 장소가 없으면 안내 문구 표시.
- **`vite.config.js`**: `npm run dev`(`localhost:5173`)에서도 `/api/*`가 로컬 API 서버(`127.0.0.1:5175`)로 연결되도록 프록시 추가.
- 결과: `map.kakao.com/link/search/...` 검색 결과 없음 페이지로 이동하는 장소 링크를 화면에서 제거.

## 2026-07-06 — 성향 결과 이미지 잘림 수정

성향 결과 화면의 일러스트가 원형 마스크(`rounded-full`)와 `object-cover`로 다시 잘리면서, 이미 둥근 사각형으로 제작된 이미지의 가장자리 요소가 답답하게 잘려 보이는 문제를 수정했다.

- **`src/App.jsx`**: 성향 결과 이미지 스타일을 원형 크롭에서 둥근 사각형(`rounded-[30px]`) + `object-contain`으로 변경.
- 결과: 미식/호캉스/알뜰 성향 일러스트가 원본 비율과 모서리 형태를 유지하며 표시되고, 음식·소품 가장자리 잘림이 사라짐.

## 2026-07-02 — 모바일 최적화 · API 검증 · 안정성

토스 인앱 웹뷰(모바일) 출시를 목표로 결과 화면 재구성, 이미지 경량화, 외부 API 점검, 실패 대응을 진행했다.

### 1. 결과 화면 모바일 재구성 (`src/App.jsx`)
- 코스 카드 높이 **1,874px → 830px (약 56%↓)**, 실제 코스(장소 리스트)가 첫 화면에 노출되도록 순서 재배치.
- 시안에 없던 중복 블록 **"AI 추천 흐름(슬롯)" 제거** (아래 실제 장소 리스트와 내용 중복이었음).
- **"AI 예산 진단"을 아코디언(기본 접힘)** 으로 변경 — 필요할 때만 펼침.
- 배치 순서: 제목 → 예상비용 → 비율 바 → 일차 탭 → 코스 동선(장소 리스트) → 지도 → 이동 안내 → AI 예산 진단(접힘).

### 2. 이미지 경량화 (`src/assets/`)
- 성향 결과 3종 + 스플래시 이미지를 **PNG → WebP** 변환 + 표시 크기 3배 해상도로 리사이즈.
- 합계 **약 5.4MB → 약 90KB (98%↓)**. `App.jsx`의 import를 `.webp`로 교체, 사용하지 않는 PNG 원본 삭제.

### 3. 외부 API 3종 검증 (실제 키로 호출)
- **TourAPI(한국관광공사 KorService2)**: ✅ 정상. 4개 지역(강릉·부산·전주·제주) 각 실제 장소 40개.
  - 최초 점검 시 data.go.kr이 일시적으로 503 장애 상태였음 → 이후 복구 확인. 키/코드/엔드포인트 모두 정상.
- **Kakao Local**: ✅ 정상. 실제 도로명주소·전화번호 보강.
- **OpenAI**: ✅ 정상. AI 예산 진단 문구 실시간 생성.

### 4. 안정성 보강
- **`server/tourApi.js`**: data.go.kr 5xx/네트워크 오류 시 백오프로 **최대 2회 자동 재시도**(`fetchWithRetry`) 추가. 최종 실패 시 서버 로그 기록.
- **`src/App.jsx`**: 결과 카드에 **데이터 출처 배지** 추가 — 실시간(TourAPI) 반영 시 "실시간 장소 반영", 실패 시 "샘플 데이터" 표시.

### 5. 후속 처리 (같은 날)
- **당일치기 호캉스 코스 제거** (`src/lib/courses.js`, `src/App.jsx`): 당일치기는 숙박 축(L=호캉스)을 코스에서 제외해 F/A 2개만 생성. 세그먼트 탭도 코스 개수에 맞춰 동적 컬럼으로.
- **문서 AI 엔진 표기 정리** (기획서 2부·`design_handoff/README.md`·`src/lib/courses.js` 주석): 런타임 "Claude API" 언급을 실제 구현(**OpenAI gpt-4.1-mini**)에 맞춰 수정. **AI 엔진은 OpenAI 유지(교체 아님).** 단, §11 "Claude Design"은 시안 제작에 쓴 도구 언급이라 사실이므로 유지.

### 6. 실제 지도 · 전국 지역 선택
- **Kakao Maps JS 실제 지도**(`src/App.jsx`): 결과 화면 placeholder → 실제 카카오 지도(TourAPI 좌표로 번호 마커+fitBounds). `VITE_KAKAO_MAP_KEY`(도메인 등록 공개키), 키/좌표 없으면 placeholder fallback.
- **전국 지역 선택**: 4개 하드코딩 → 전국 시/도 17 + 시/군/구 234개.
  - `scripts/generate-regions.mjs`로 TourAPI에서 코드 트리 생성 → `src/data/regions.json` (시/도명 정규화: 경기도→경기 등).
  - `server/tourApi.js`: 하드코딩 REGION_CODES → regions.json 기반 지역명→코드 매핑.
  - `src/App.jsx`: 단일 select → **바텀시트 드릴다운 선택기**(시/도→시/군/구 + 전국 검색). region 문자열 형식 유지로 지도·공유 호환.
  - 검증: 경기 평택/서울 강남/충남 태안 실데이터+지도+AI 확인.

### 참고
- 실제 API가 반영된 화면은 `npm run build && npm run serve`(포트 5175)로 확인. vite dev(5174)는 `/api/*`를 프록시하지 않아 항상 샘플 데이터.
- 서버 코드(`server/*.js`) 수정 시 실행 중인 serve 프로세스는 재시작해야 반영됨(지역 매핑 등).
