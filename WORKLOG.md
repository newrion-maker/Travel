# 작업 로그

## 2026-07-02 — 모바일 최적화 · API 검증 · 안정성

토스 인앱 웹뷰(모바일) 출시를 목표로 결과 화면 재구성, 이미지 경량화, 외부 API 점검, 실패 대응을 진행했다.

### 1. 결과 화면 모바일 재구성 (`src/App.jsx`)
- 코스 카드 높이 **1,874px → 830px (약 56%↓)**, 실제 코스(장소 리스트)가 첫 화면에 노출되도록 순서 재배치.
- 시안에 없던 중복 블록 **"AI 추천 흐름(슬롯)" 제거** (아래 실제 장소 리스트와 내용 중복이었음).
- **"AI 예산 진단"을 아코디언(기본 접힘)** 으로 변경 — 필요할 때만 펼침.
- 배치 순서: 제목 → 예상비용 → 비율 바 → 일차 탭 → 코스 동선(장소 리스트) → 지도 → 이동 안내 → AI 예산 진단(접힘).

### 2. 이미지 경량화 (`src/assets/`)
- 성향 결과 3종 + 스플래시 이미지를 **PNG → WebP** 변환 + 표시 크기 3배 해상도로 리사이즈.
- 합계 **약 5.4MB → 약 90KB (98%↓)**. `App.jsx`의 import를 `.webp`로 교체, 사용하지 않는 PNG 원본 삭제.

### 3. 외부 API 3종 검증 (실제 키로 호출)
- **TourAPI(한국관광공사 KorService2)**: ✅ 정상. 4개 지역(강릉·부산·전주·제주) 각 실제 장소 40개.
  - 최초 점검 시 data.go.kr이 일시적으로 503 장애 상태였음 → 이후 복구 확인. 키/코드/엔드포인트 모두 정상.
- **Kakao Local**: ✅ 정상. 실제 도로명주소·전화번호 보강.
- **OpenAI**: ✅ 정상. AI 예산 진단 문구 실시간 생성.

### 4. 안정성 보강
- **`server/tourApi.js`**: data.go.kr 5xx/네트워크 오류 시 백오프로 **최대 2회 자동 재시도**(`fetchWithRetry`) 추가. 최종 실패 시 서버 로그 기록.
- **`src/App.jsx`**: 결과 카드에 **데이터 출처 배지** 추가 — 실시간(TourAPI) 반영 시 "실시간 장소 반영", 실패 시 "샘플 데이터" 표시.
